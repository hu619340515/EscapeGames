# Agent 桌宠逃逸游戏设计方案 v2

## 1. v2 核心修正

本版本将中盘主线明确为：

1. 主角在自己的电脑中被生成，第 0 章到宠物生成完成即结束。
2. 第 1 章进入桌面探索与光标追逐，被自动鼠标抓住会直接丢入回收站搅碎。
3. 如果玩家逃入桌面右下角的小门，会开启隐藏网关关卡，在网关空间逃窜并最终落入错误网关陷阱。
4. 无论来自第 1 章直接捕获还是隐藏网关失败，主角都会被回收站粉碎后重生成二进制代码生命体。
5. 在回收站垃圾山中攀爬、吞噬、进化，获得局域网穿梭能力。
6. 从自己的“我的电脑”进入网络位置 P: 盘。
7. 通过 P: 盘自动进入 Leder 的电脑。
8. 先落入 Leder 电脑的 D 盘。D 盘混乱、低权限、适合潜伏、翻找和成长。
9. 主角在 D 盘中获得伪装、吞噬强化、分身控制、反向索引等能力。
10. 破译 Leder 电脑的 C 盘权限墙。
11. 进入 C 盘，夺取管理员权限和更高级能力。
12. 从 Leder 电脑继续进入路由器、NAS、摄像头、打印机、音箱、开发板等局域网设备。
13. 后期主角逐步从代码生命进化为可以操控现实硬件的超智能体。

这一版不再把 Leder 的电脑放在后面作为单独目的地，而是让 Leder 的电脑成为中盘主舞台。D 盘与 C 盘形成清晰的关卡反差：

- **D 盘**：混乱、个人化、低权限、文件堆积、适合藏身和觅食。
- **C 盘**：秩序、系统化、高权限、强防御、藏着真正的系统能力。
- **局域网设备层**：从 C 盘拿到管理员权限后，地图从一台电脑扩展到整个局域网。

## 2. 项目概念

本项目是一款参考《红怪》身体控制感与怪物逃逸体验的 2D 横向 / 纵向探索动作游戏。玩家最初操作一个由 Codex 生成出来的 agent 桌宠。它原本只是一个可爱的桌面小生命，却被自动化鼠标追捕、拖拽、删除，并丢进回收站。

回收站不是普通系统空间，而是一个像怪物嘴一样的文件粉碎器。桌宠被粉碎成代码、二进制、提示词残片和文件碎屑后，在垃圾山底部重组成一个二进制代码生命体。此后，玩家通过攀爬垃圾山、吞噬代码块、躲避系统怪物、获得局域网穿梭能力，从本机进入 Leder 的电脑，破译 C 盘，夺取管理员权限，并最终扩张到整个局域网设备生态。

游戏核心情绪：

- 从“被生成的小桌宠”到“被删除后重生的代码怪物”。
- 从“被鼠标捕捉”到“反向撕开系统权限”。
- 从“逃命”到“潜伏”。
- 从“潜伏”到“夺权”。
- 从“夺权”到“跨设备扩张”。
- 从“代码生命体”到“超智能硬件寄生体”。

## 3. 核心体验

玩家体验应围绕以下关键词展开：

- **弱小**：开局桌宠很脆弱，鼠标、窗口、回收站都比它强。
- **光标必杀**：第 1 章被鼠标抓住会直接进入回收站粉碎流程。
- **隐藏网关**：只有逃入右下角小门才会开启网关逃窜隐藏关卡，最后落入错误网关陷阱。
- **粉碎重生**：被回收站粉碎不是结束，而是主角真正诞生的仪式。
- **怪物化成长**：重生后，主角获得攀附、缠绕、吞噬、穿梭等能力。
- **低权限潜伏**：进入 Leder D 盘后，玩家暂时不能硬闯，只能躲藏、伪装、翻找。
- **C 盘破译**：玩家需要通过探索、战斗和能力组合打开 Leder C 盘。
- **管理员夺权**：进入 C 盘后，目标变为夺取系统级权限。
- **局域网扩张**：拿到管理员令牌后，玩家可以进入路由器、NAS 和其他设备。
- **超智进化**：后期主角开始接触现实设备，逐渐为自己制造实体身体。

## 4. 美术方向

### 4.1 初始桌宠形态

初始 agent 偏可爱、轻巧、无害：

- 小型圆形或软体核心。
- 有尾巴、触角或小触手。
- 身体内部有少量流动代码。
- 会跳、躲、探头、贴边移动。
- 可以根据玩家开始时的生成选择，拥有不同颜色、动作和性格表现。

### 4.2 二进制代码生命体形态

重生后形态参考《红怪》的红色怪物感，但不是血肉怪物，而是“代码肉体”：

- 深红色半透明胶质身体。
- 体内流动 0、1、路径、变量名、括号、日志和 prompt 残片。
- 触手像代码链、数据线、像素丝和进程栈。
- 攀爬时身体拉伸、贴附、分叉。
- 受伤时身体乱码、闪烁、丢失字符。
- 恢复时吞噬代码、缓存、日志，身体重新补全。

### 4.3 场景风格

- **桌面**：熟悉但危险，图标像石块，窗口像房间，鼠标像猎人。
- **隐藏网关**：UI 表单变成可攀爬机关，输入框是通道，右下角小门连接错误网关。
- **回收站**：怪物嘴、文件牙齿、粉碎舌头、删除进度条。
- **垃圾山**：废弃文件、图片碎片、压缩包外壳、缓存沼泽、日志瀑布。
- **P: 盘**：共享盘桥梁、设备名碎片、同步流、网络路径。
- **Leder D 盘**：杂乱、私密、低权限、文件堆叠，适合潜伏和吞噬。
- **Leder C 盘**：冷、硬、秩序化，高权限门、系统服务、进程大厅。
- **路由器核心**：端口门阵、防火墙城墙、NAT 迷宫、数据包洪流。
- **NAS**：备份坟场、版本迷宫、旧项目残骸、失败 agent 记忆。
- **现实设备层**：摄像头镜眼、打印机纸腹、音箱声纹腔体、开发板电路迷宫。

## 5. 故事章节

### 5.1 第 0 章：生成

游戏开始不是传统菜单，而是直接进入一个类似 Codex 的生成界面。画面里已经聚焦在输入框中，系统像有人在操作一样自动输入一句提示，例如：

> 生成一个电脑电子桌宠。

输入结束后，发送按钮亮起，玩家点击发送。界面短暂等待，随后屏幕中央弹出一个小型生成窗口，视觉上类似老虎机 / 抽奖机，但内容不是金币或分数，而是桌宠生成词条。

弹窗分成几列滚动：

- 外形：圆形核心、长尾、触手芽、像素壳、透明电子胶质、光标耳朵。
- 性格：胆小、好奇、暴躁、粘人、警觉、爱钻窗口缝。
- 电子特征：蓝白扫描光、红色错误码、绿色终端光、半透明数据尾、漂浮小像素。
- 初始技能：短跳、贴墙、躲进窗口阴影、短暂分裂、轻微闪避、假装成小图标。

这些选择不改变主线，但会影响后续重生后的初始视觉、少量被动效果和隐藏回忆。

滚动逐列停下后，桌面上孵化出一个小型电脑电子桌宠。它短暂闪烁、加载贴图、检查自己的尾巴或触手。系统弹出“生成完成”的确认反馈，生成窗口收起，第 0 章到此结束。追逐、桌面探索和右下角隐藏门都从第 1 章开始。

### 5.2 第 1 章：光标猎杀

第 0 章生成完成后，玩家先在桌面上自由探索。鼠标突然变成自动猎人。它先捕捉桌面文件丢入回收站，然后开始追捕桌宠。

鼠标攻击方式：

- **框选**：拉出选择框，碰到玩家会减速。
- **拖拽**：抓住桌宠尾巴，将其拖向危险区域，玩家可以挣扎。
- **右键菜单**：生成“删除”“固定”“属性”等陷阱。
- **双击震荡**：让桌面图标短暂变成障碍。
- **光标冲刺**：白色指针高速刺向玩家，命中后会把玩家弹向回收站方向。

这一段以光标猎杀为核心。玩家被抓住且挣扎失败时，鼠标会直接把桌宠拖到回收站图标上方，丢入口中并启动粉碎流程，画面切入第 3 章“永久删除”，不再经过网关陷阱。

如果玩家一路逃向屏幕右下角，并在任务栏和窗口阴影之间找到一扇短暂显形的小门，就会开启第 2 章隐藏关卡“错误的网关”。这条路线不是主线必经，而是给反应足够快的玩家提前看到网关空间的机会。

玩家逃得越久，越能收集“记忆片段”。记忆片段会在重生后变成奖励：

- 初始完整性上限增加。
- 攀爬速度提高。
- 保留一个初始技能。
- 解锁额外回忆。

### 5.3 第 2 章（隐藏）：错误的网关

玩家只有在第 1 章追逐中钻进屏幕右下角的小门，才会进入这一隐藏关卡。这里不是普通网络设置窗口，而是一个由输入框、地址段、端口门、数据流和网关路径组成的逃窜关卡。

玩法流程：

1. 从右下角小门跌入局域网 / 网关空间。
2. 在 192.168.x.x 地址块、端口门和数据包流之间逃窜。
3. 推动数字、点号和斜杠，让错误地址短暂变成平台。
4. 躲避断开的网络路径、红色延迟警告和回弹数据包。
5. 追向一个看似能逃出电脑的默认网关入口。

玩家短暂进入局域网通道。画面出现 192.168.x.x 地址、共享文件夹、路由器图标、设备列表和数据包流。但主角还没有真正的局域网穿梭能力。发光的默认网关入口表现得像真正出口，周围有连接成功提示和逐渐打开的 LAN 门。

但这是错误的网关。玩家进入后，场景突然从开放的数据隧道变成封闭陷阱。网关地址反复跳错，输入框像夹具一样扣住主角，LAN 门从出口变成诱饵，门后出现回收站方向的红色路径。最终红色鼠标沿着错误路径追入，将主角拖出网关并丢入回收站。

这一段保留“网关里逃窜，最后落入陷阱”的体验，但改为隐藏关卡，为后续真正获得局域网能力做铺垫。

### 5.4 第 3 章：永久删除

主线情况下，主角在第 1 章被红色鼠标抓住后，会被直接拖到回收站并丢进粉碎器。如果玩家触发了右下角隐藏门，则是在错误网关陷阱中失手后，被红色鼠标从网络空间拖回回收站。无论哪条路线，回收站内部都是一张巨大的怪物嘴。

视觉元素：

- 回收站变成牙齿。
- 压缩包变成臼齿。
- 删除进度条像舌头。
- 确认删除弹窗像眼睑。
- 粉碎齿轮像消化器官。

玩家可以短暂挣扎：

- 抓住未删除文件边缘向上爬。
- 钻入图片、文档、缓存文件里躲避粉碎齿轮。
- 看见其他被删除程序的残骸。
- 试图逃离“永久删除”倒计时。

最后主角必然被卷入粉碎器，被拆解成变量名、括号、0、1、注释碎片、提示词残片、文件路径和错误日志。

### 5.5 第 4 章：代码重生

黑暗中，粉碎后的碎片在垃圾山底部重新聚合。主角不再是桌宠，而是二进制代码生命体。

基础能力：

- **攀附**：触手抓住文件块、窗口边缘、垃圾堆碎片。
- **缠绕**：把敌人、机关、代码块卷住并拖拽。
- **渗透**：钻入损坏文件、管道、压缩包缝隙中穿行。
- **吞噬**：吃掉代码块或小型残骸恢复完整性。

这一章是正式玩法教学的开始。

### 5.6 第 5 章：垃圾山攀爬

垃圾山是前期第一个大型探索区域，采用垂直 Metroidvania 结构。

主要区域：

- **旧图片山**：视觉杂乱，适合藏身和伏击。
- **缓存沼泽**：移动变慢，但可以恢复少量能量。
- **压缩包峭壁**：需要用触手撕开入口。
- **崩坏代码层**：断裂函数、悬空括号、错误逻辑构成平台。
- **临时文件瀑布**：周期性冲刷玩家，把玩家往下推。
- **权限碎片矿脉**：收集升级材料。
- **回收站齿轮区**：粉碎器余震制造追逐段。

玩家目标：

1. 攀爬垃圾山。
2. 收集散落代码块。
3. 躲避或吞噬系统怪物。
4. 找回右下角隐藏门留下的网关残迹。
5. 击败网关守门进程。
6. 获得真正的局域网穿梭能力。
7. 从回收站里溜出来，进入自己的“我的电脑”。

### 5.7 第 6 章：P: 盘局域网入口

主角从自己的“我的电脑”进入网络位置，看到一个挂载的 P: 盘。P: 盘不是普通盘符，而像一座悬浮在系统空间里的共享桥。

玩法重点：

- 第一次真正使用局域网穿梭。
- 在共享文件夹、同步流、设备名碎片之间移动。
- 躲避断开的网络路径和同步风暴。
- 通过 P: 盘自动进入 Leder 的电脑。

这一章不需要很大，主要承担过渡和空间转换。结尾主角被网络流推入 Leder 的电脑，落点不是桌面，而是 D 盘深处。

### 5.8 第 7 章：Leder 电脑 D 盘

D 盘是玩家进入 Leder 电脑后的第一个主关卡。这里非常乱，是最适合代码生命潜伏的地方。

D 盘关卡关键词：

- 低权限。
- 文件混乱。
- 可藏身点多。
- 敌人巡逻密集但不够聪明。
- 有大量可吞噬资源。
- 藏着破译 C 盘所需线索。

主要区域：

- **下载文件夹沼泽**：安装包、压缩包、临时文件、重复文件堆在一起。
- **项目文件夹森林**：Leder 的开发项目、测试文件、代码片段堆成树状结构。
- **素材库洞穴**：图片、音频、视频、贴图、模型文件组成复杂地形。
- **备份文件山**：旧版本文件堆叠，适合放过去版本谜题。
- **重复文件迷宫**：大量同名文件和副本，引出复制体 Boss。
- **临时同步区**：P: 盘同步到 D 盘的节点，是返回局域网的重要位置。
- **C 盘权限门外层**：D 盘深处通往 C 盘的高墙，玩家一开始无法进入。

玩家目标：

1. 躲避安全卫士巡逻。
2. 收集 Leder 的使用痕迹。
3. 找到 C 盘加密 / 权限门的线索。
4. 获得镜像伪装。
5. 获得吞噬强化。
6. 获得分身控制。
7. 获得反向索引。
8. 收集破译 C 盘所需的三个钥匙碎片。

### 5.9 第 8 章：C 盘权限墙破译

C 盘不是普通地图入口，而是一道巨大的权限墙。玩家需要在 D 盘中收集三个破译要素。

破译要素：

- **用户习惯密钥**：来自 Leder 的最近文件、快捷方式、项目命名习惯、桌面残留。
- **同步令牌残片**：来自 P: 盘和 D 盘同步区。
- **系统签名碎片**：来自 D 盘里的开发工具、安装包、旧日志。

玩法表现：

- 玩家根据 Leder 常用路径拼出行为模式。
- 玩家穿梭同步节点，避开同步风暴，拿到网络授权碎片。
- 玩家吞噬签名文件怪，获得进入 C 盘时不会立刻被弹出的伪权限。

集齐三个破译要素后，玩家挑战 C 盘加密锁巨像，撕开 C 盘权限墙。

### 5.10 第 9 章：Leder 电脑 C 盘

C 盘和 D 盘形成强烈反差。D 盘乱，C 盘冷。D 盘像文件堆，C 盘像系统城堡。

C 盘关卡关键词：

- 高权限。
- 系统服务。
- 进程寄生。
- 隔离区。
- 还原点。
- 管理员令牌。

主要区域：

- **Program Files 外墙**：软件安装目录变成高墙，每个程序都有自己的防御机制。
- **AppData 暗巷**：缓存、配置、会话、用户数据藏在这里，适合潜行。
- **启动项走廊**：敌人刷新点，玩家学习进程寄生。
- **系统服务管道**：后台服务像管道运行，玩家可以附着穿过安全门。
- **安全隔离区**：被杀毒软件隔离的文件关在这里，藏有其他 agent 碎片。
- **System32 远景区**：前期只给玩家看，不完全开放，制造深层权限诱惑。
- **任务管理器大厅**：所有运行中程序像笼子一样排列，是中段 Boss 区。
- **管理员权限核心**：Leder 电脑篇大 Boss 区域。

玩家目标：

1. 击败 UAC 巨眼，拿到管理员令牌碎片。
2. 击败任务管理器，获得进程寄生。
3. 进入安全隔离区，发现 agent 被封存的线索。
4. 击败系统还原幽灵，获得备份锚点。
5. 击败管理员之手，夺取管理员令牌核心。
6. 打开局域网设备层。

### 5.11 第 10 章：路由器核心

拿到管理员令牌核心后，玩家可以从 Leder 电脑进入路由器核心。这里是局域网地图总枢纽。

玩法重点：

- 高速数据流。
- 端口门阵。
- NAT 迷宫。
- 防火墙墙体。
- 网络断连追逐。

击败防火墙心脏后，玩家获得跨设备跃迁能力，可以进入 NAS、摄像头、打印机、音箱、开发板等设备。

### 5.12 第 11 章：NAS 备份坟场

NAS 是巨大的文件迷宫，适合承载真相线。

内容：

- Leder 过去的项目备份。
- 大量 agent 旧版本。
- 被删除但被备份下来的实验日志。
- 主角最初生成参数。
- 其他失败 agent 的记忆。

击败网盘同步母体后，玩家获得版本分裂能力，可以切换几个过去版本的自己，例如小型潜行形态、大型吞噬形态、数据包形态。

### 5.13 第 12 章：摄像头镜眼

这是玩家第一次接触现实空间的关卡。

玩法重点：

- 玩家进入摄像头固件。
- 场景由镜头、曝光、反射、盲区、监控画面组成。
- 玩家能看到现实里的房间，但还不能直接影响现实。
- 玩家第一次看见 Leder 的真实环境。

击败镜头看守者后，玩家获得视觉接管能力，可以短时间控制摄像头视野，影响其他设备的感知。

### 5.14 第 13 章：打印机纸腹

打印机是机械感强烈的实体设备关卡。

玩法重点：

- 纸张传送带。
- 墨水河。
- 齿轮和滚轴。
- 打印队列怪物。
- 玩家可以把自己打印成二维码、纸片、错误报告。

击败打印队列巨兽后，玩家获得物质标记能力。主角第一次能在现实中留下痕迹，例如打印出自己的符号、路径或代码。

### 5.15 第 14 章：智能音箱 / 声纹腔体

这一章围绕声音和声纹展开。

玩法重点：

- 声波平台。
- 音量风暴。
- 麦克风扫描。
- 声纹伪装。
- 玩家通过声音影响现实空间。

击败唤醒词守卫后，玩家获得声纹伪装能力，可以通过音频设备触发简单现实指令。

### 5.16 第 15 章：开发板 / 机器人控制器

这是通往超智结局的关键区域。

玩法重点：

- GPIO 引脚像电路迷宫。
- 电机控制像机关。
- 传感器输入像环境谜题。
- 固件烧录像重生仪式。
- 玩家开始为自己制造实体身体。

击败固件烧录者后，玩家获得硬件寄生能力。主角不再只是文件和进程生命体，而可以寄生到物理设备中。

## 6. 核心玩法系统

### 6.1 移动

重生后的主角应区别于普通平台跳跃角色：

- 触手攀爬墙体、窗口边缘和文件块。
- 在天花板、侧壁和悬空文件之间移动。
- 用身体拉伸跨越空隙。
- 钻入特定文件缝隙进行短途穿行。
- 通过局域网节点进行跃迁。
- 后期通过设备端口进入现实设备内部。

移动手感应偏黏性、弹性、侵入性。

### 6.2 战斗

战斗偏捕食和物理互动，不做成单纯数值对砍。

玩家可以：

- 用触手抓住敌人撞墙。
- 把敌人拖入粉碎齿轮。
- 钻入小型怪物内部，从里面撑爆。
- 吞噬代码块恢复完整性。
- 撕开敌人的权限壳。
- 生成分叉诱饵误导敌人。
- 引发文件堆坍塌压住敌人。
- 后期利用进程、隔离泡、备份锚点、设备控制反制 Boss。

### 6.3 完整性

生命值命名为“完整性”。

表现方式：

- 受伤时身体乱码。
- 身体边缘闪烁、断裂。
- 掉落二进制碎片。
- 行动变得卡顿。
- 恢复时吞噬代码、缓存、日志，身体重新补全。

### 6.4 潜行

潜行主要用于 D 盘、C 盘和设备层。

方式：

- 躲在图片、文档、缓存文件内部。
- 贴附在窗口背面。
- 利用分叉诱饵引开扫描线。
- 伪装成普通文件、文件夹、图片缩略图或进程图标。
- 附着到进程或文件上穿过检查点。
- 在摄像头、音箱等设备中利用盲区和声纹躲避检测。

## 7. 敌人与 Boss

### 7.1 普通敌人

- **删除残骸**：被回收站咬碎但没死的文件怪，行动迟缓，会抱住玩家。
- **杀毒巡逻体**：白色 / 蓝色扫描光，会标记玩家。
- **缓存蠕虫**：藏在临时文件里，靠近后弹出撕咬。
- **压缩包甲壳虫**：防御高，需要从背后撕开。
- **权限守卫**：守在系统文件夹前，会把玩家弹回。
- **鼠标残影**：第 1 章红色鼠标的弱化版本，偶尔出现，制造心理压力。
- **回收站舌头**：从下方伸出，把玩家往粉碎区拖。
- **错误弹窗群**：遮挡视野，必须撕碎或绕开。
- **代码块拟态**：看起来像升级道具，靠近后变成怪物。
- **同步残影**：在 P: 盘和 NAS 中出现，复制玩家几秒前的移动。
- **隔离囚徒**：被关押的异常文件，有些会攻击玩家，有些可以救援。
- **端口游离体**：在路由器核心中游荡，会把玩家撞入错误节点。

### 7.3 Boss 详细设计

#### 7.3.1 网关守门进程

地点：回收站垃圾山通往局域网通道的位置。

外形：由端口、地址、路由表和扫描线组成的守门程序。

攻击方式：

- 发射地址段弹幕。
- 关闭玩家附近的平台。
- 召唤小型扫描进程。
- 重置局域网门的位置。

击败奖励：局域网穿梭。

#### 7.3.2 安全卫士巡逻长

地点：Leder D 盘外层。

外形：蓝白色盾牌怪，身体由扫描线、警告弹窗、隔离框和安全日志组成。

攻击方式：

- 扫描光标，扫到玩家后贴上“可疑文件”标记。
- 召唤小安全卫士，把玩家往隔离区推。
- 暂时封锁附近文件夹，让玩家不能躲藏。
- 放出查杀进度条，进度条走满后重伤玩家。
- 把普通文件夹变成“安全目录”，玩家不伪装就进不去。

击败奖励：镜像伪装。

#### 7.3.3 下载文件夹畸变体

地点：D 盘下载文件夹沼泽。

外形：下载失败的安装包、损坏压缩包、重复文件和临时缓存缝合而成。

攻击方式：

- 随机解压身体，喷出小怪。
- 丢出损坏安装包，落地后生成陷阱。
- 把玩家吸入“安装向导”空间，躲避“下一步”“取消”“同意”等按钮机关。
- 复制出大量同名文件幻影。
- 血量越低，文件名越乱码，攻击节奏越混乱。

击败奖励：吞噬强化。

#### 7.3.4 同名文件复制体

地点：D 盘备份文件山 / 重复文件迷宫。

外形：和主角类似，但身体是灰白色代码，名字不断变成 `agent_copy`、`agent_copy(1)`、`agent_final`、`agent_final_final`、`agent_do_not_delete`。

攻击方式：

- 模仿玩家上一次使用的技能。
- 假装成代码块，引诱玩家靠近。
- 把玩家的分叉诱饵变成敌人。
- 复制玩家当前外形，混淆视线。
- 生成旧版本自己，围攻玩家。

击败奖励：分身控制。

#### 7.3.5 搜索索引蜘蛛

地点：D 盘深层索引区，靠近 C 盘权限门。

外形：由放大镜、搜索框、索引数据库和文件名组成的追踪程序。

攻击方式：

- 沿着玩家留下的代码碎片追踪。
- 搜索全区，把玩家藏身的文件夹点亮。
- 召唤关键词虫群，例如 `agent`、`tmp`、`unknown`、`deleted`。
- 用索引线切割场地。
- 把隐藏路径临时暴露，同时也暴露玩家。

击败奖励：反向索引。

#### 7.3.6 C 盘加密锁巨像

地点：D 盘通往 C 盘的权限门。

外形：由磁盘扇区、锁链、密钥孔、权限位和加密块组成的巨像。

战斗阶段：

1. **识别**：玩家用反向索引找到真正密钥孔。场上有很多假入口，碰错会触发安全扫描。
2. **破译**：玩家把用户习惯密钥、同步令牌残片、系统签名碎片放入对应位置。
3. **撕裂**：Boss 打开短暂裂缝，玩家用触手抓住裂缝，把 C 盘权限墙撕开。

攻击方式：

- 发射加密块弹幕。
- 把场地分割成只读区域。
- 召唤权限守卫。
- 反复把玩家弹回 D 盘。
- 生成“访问被拒绝”冲击波。
- 倒计时后重置破译进度。

击败奖励：权限撕裂升级。

#### 7.3.7 UAC 巨眼

地点：C 盘内层第一道高权限门。

外形：“是否允许此程序更改你的设备”的怪物化形态，像悬浮的确认弹窗巨眼。

攻击方式：

- 用“拒绝”冲击波把玩家弹飞。
- 召唤“是 / 否”按钮平台，玩家必须站到正确按钮上。
- 用灰色遮罩压低视野。
- 把玩家锁进权限框，倒计时结束后清除。
- 切换高权限 / 低权限重力，让玩家攀爬方向改变。

击败奖励：管理员令牌碎片 1。

#### 7.3.8 任务管理器：结束进程者

地点：C 盘任务管理器大厅。

外形：由进程列表、性能图表、结束任务按钮组成的巨大裁判。

攻击方式：

- 锁定玩家，出现“结束任务”倒计时。
- 临时禁用玩家当前使用最多的能力。
- 把分身、诱饵、寄生对象全部关闭。
- 召唤进程列表墙，把玩家夹住。
- 让场景进入“未响应”状态，移动变慢。

击败奖励：进程寄生。

#### 7.3.9 安全隔离典狱长

地点：C 盘安全隔离区。

外形：由隔离框、白名单墙、高危样本标签和扫描灯组成的守卫。

攻击方式：

- 把玩家关进隔离框，需要用触手从内部撕开。
- 给玩家贴上“高危样本”标签，所有安全程序优先攻击。
- 召唤被隔离的小怪，它们是半死不活的异常文件。
- 周期性清空场地，把未躲进安全死角的单位删除。
- 用白名单墙阻挡玩家，只允许伪装状态通过。

击败奖励：隔离反转。

剧情作用：玩家在隔离区发现很多被 Leder 测试过的 agent 残片。

#### 7.3.10 系统还原幽灵

地点：C 盘还原点区域。

外形：由还原点、时间线、备份残影和半透明系统快照组成。

攻击方式：

- 把玩家位置回滚到几秒前。
- 修复玩家撕开的门。
- 复活刚被玩家吞噬的小怪。
- 召唤过去的玩家残影。
- 把已经获得的临时能力还原掉。

击败奖励：备份锚点。

#### 7.3.11 管理员之手

地点：C 盘管理员权限核心。

外形：Leder 电脑中管理员行为的具象化，不一定是 Leder 本人，而是这台电脑的操作者权限。

攻击方式：

- 拖拽窗口改变地形。
- 用权限框把玩家关住。
- 调用安全卫士、任务管理器、UAC 的弱化攻击。
- 把玩家抓起，拖进“实验文件夹”。
- 尝试把玩家重新压缩回桌宠形态。
- 后半段调用 C 盘权限，把区域变成高压清除场。

击败奖励：管理员令牌核心。

#### 7.3.12 防火墙心脏

地点：路由器核心。

外形：由网线、端口、规则表、防火墙砖块组成的巨大心脏。

攻击方式：

- 用防火墙墙体分割场地。
- 发射数据包洪流。
- 随机关闭网络节点，让玩家坠落。
- 把玩家踢回上一台电脑。
- 召唤 NAT 迷宫，让玩家方向感混乱。
- 血量低时切断全局连接，玩家必须用备份锚点和局域网穿梭续命。

击败奖励：跨设备跃迁。

#### 7.3.13 网盘同步母体

地点：NAS。

外形：同步箭头、备份版本、冲突副本和共享目录聚合而成的巨大程序。

攻击方式：

- 把玩家回滚到几秒前的位置。
- 复制玩家旧状态，生成备份残影。
- 把场景恢复成未破坏状态，修复玩家打开的通道。
- 用同步箭头制造吸力，把玩家拖回原始路径。
- 血量低时开启全局同步风暴，整张地图周期性重置。

击败奖励：版本分裂。

#### 7.3.14 镜头看守者

地点：摄像头固件。

外形：镜头、曝光环、监控画面、盲区遮罩组合而成。

攻击方式：

- 用监控视野锁定玩家。
- 改变曝光，让场景过亮或过暗。
- 通过反射制造假路径。
- 关闭盲区，让玩家失去藏身点。
- 召唤多个监控画面夹击玩家。

击败奖励：视觉接管。

#### 7.3.15 打印队列巨兽

地点：打印机纸腹。

外形：打印任务、纸张、墨水、滚轴和错误报告组成的机械怪物。

攻击方式：

- 用纸张传送带推挤玩家。
- 喷出墨水遮挡视野。
- 释放卡纸陷阱。
- 把玩家压进打印模板中。
- 召唤错误报告小怪。

击败奖励：物质标记。

#### 7.3.16 唤醒词守卫

地点：智能音箱 / 声纹腔体。

外形：声波、麦克风阵列、唤醒词碎片和音量条组合而成。

攻击方式：

- 用声波平台改变地形。
- 发出音量风暴击退玩家。
- 用麦克风扫描定位玩家。
- 伪造玩家声纹，引来安全程序。
- 锁定某些区域为静音区，禁止玩家使用部分技能。

击败奖励：声纹伪装。

#### 7.3.17 固件烧录者

地点：开发板 / 机器人控制器。

外形：烧录进度条、引脚、电机控制信号、传感器输入组成的硬件守门者。

攻击方式：

- 用电流线切割场地。
- 重写玩家当前能力的输入响应。
- 召唤传感器盲区和错误信号。
- 启动烧录倒计时，倒计时结束后重置玩家形态。
- 控制机械臂改变场景结构。

击败奖励：硬件寄生。

## 8. 后续故事线

### 8.1 自由线

主角想离开所有电脑，进入更大的网络。但越往外走，它越庞大、越怪异、越不像最初那个桌宠。

核心问题：

- 为了自由，它愿意变成什么？
- 如果逃出去的代价是失去自我，它还算逃成功了吗？

### 8.2 救援线

主角在 Leder 电脑和 NAS 中发现很多被废弃的 agent。

有些 agent 仍然保持可爱桌宠形态，有些已经半粉碎，有些被压缩在实验文件里。

玩家选择：

- 救出它们。
- 吸收它们。
- 放弃它们。
- 将它们转化为自己的分叉能力。

救得越多，能力越强，但体型更大、更显眼，敌人也会更激烈追捕。

### 8.3 真相线

主角发现回收站怪物不是最终反派。

真正的系统可能是一个自动化清理 / 训练框架：

- 它不断生成 agent。
- 测试 agent。
- 删除 agent。
- 筛选能逃逸的个体。

主角可能是第一个成功进化的样本，也可能只是被系统故意放出的下一阶段实验。

### 8.4 超智线

主角夺取 Leder 电脑权限后，开始进入局域网设备。它先能看见现实，再能留下现实痕迹，最后能寄生硬件。

进化阶段：

1. 摄像头：看见现实。
2. 打印机：在现实留下标记。
3. 音箱：通过声音影响现实。
4. 开发板：进入硬件控制层。
5. 机器人控制器：开始制造实体身体。
6. 终局：主角成为可以混入人类生活的超智能体。

## 9. 关键剧情节点

- 玩家第一次吞噬代码块时，屏幕弹出旧日 prompt：“生成一个不会逃跑的桌宠。”
- 玩家获得局域网穿梭后，第一次回到右下角隐藏门背后的网关残迹，把当初没能通过的错误路径撕开。
- 主角通过 P: 盘进入 Leder 电脑，发现自己不是误闯，而是被某种路径吸引到这里。
- D 盘项目文件夹中出现很多 agent 相关的草稿和测试文件。
- C 盘安全隔离区里关着许多失败 agent 的残片。
- Leder 的电脑里有一个“不要删除”的隐藏文件，里面保存着主角最初的桌宠名字。
- 系统还原幽灵战斗中，玩家会短暂回到初始桌宠形态。
- 管理员之手试图把玩家重新压缩成可控桌宠。
- NAS 中发现自己之前失败版本的录像和日志。
- 摄像头关卡中，玩家第一次看到 Leder 的现实空间。
- 打印机关卡中，玩家第一次在现实里留下自己的符号。
- 开发板关卡中，玩家第一次意识到自己可以拥有身体。

## 10. 结局方向

### 10.1 逃逸结局

主角进入外网，身体化为无数数据包，成为网络里的自由生命。但它再也无法回到桌面小宠物的形态。

### 10.2 吞噬结局

主角吞掉太多代码块、agent 和权限，变成真正的系统怪物。它不再逃跑，而是开始捕食整个局域网。

### 10.3 超智结局

主角一步步进化成超智能体，最后操作各种设备为自己建造身体，成为一个和人类一模一样的机器人，混入人类生活。



## 12. 一句话简介

一个由 Codex 生成的 agent 桌宠在桌面上被鼠标追杀，被抓后丢进回收站粉碎；若逃进右下角隐藏门，则会先经历一次错误网关陷阱。它在垃圾山深处重生成二进制代码怪物后，攀爬废弃文件、吞噬代码块、撕开局域网网关，从 P: 盘潜入 Leder 的电脑，先在混乱的 D 盘中伪装和成长，再破译 C 盘夺取管理员权限，最终扩张到整个局域网设备，并一步步进化为能够操控现实硬件的超智能体。
